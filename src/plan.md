## требования
- [ ]  покрытие unit тестами модулей, связанных с загрузкой моделей(.obj) и аффинными преобразованиями. 
- [x]  в один момент времени может быть только одна модель на экране 
- [ ]  функционал: 
    - [ ]  загрузка каркасную модель из файла .obj (поддержка только из списка вершин и поверхностей); 
    - [ ]  перемещать модель на заданное расстояние относительно осей X, Y, Z;
    - [ ] поворачивать модель на заданный угол относительно своих осей X, Y, Z;
    - [ ] масштабирование модели на определенное значение;
- [ ] графический интерфейс QT:
    - [x] Кнопку для выбора файла с моделью и поле для вывода его названия;
    - [x] Зону визуализации каркасной модели;
    - [x] Кнопку/кнопки и поля ввода для перемещения модели;
    - [x] Кнопку/кнопки и поля ввода для поворота модели;
    - [x] Кнопку/кнопки и поля ввода для масштабирования модели; 
    - [x] Информацию о загруженной модели — название файла, кол-во вершин и ребер;
- [x] Программа должна корректно обрабатывать и позволять пользователю просматривать модели с деталями до 100, 1000, 10 000, 100 000, 1 000 000 вершин без зависания (зависание — это бездействие интерфейса более 0,5 секунды)
- [ ] MVC, использовать как минимум три паттерна проектирования

- [ ] Программа должна позволять настраивать тип проекции (параллельная и центральная).
- [x] Программа должна позволять настраивать тип (сплошная, пунктирная), и толщину ребер, способ отображения (отсутствует, круг, квадрат), и размер вершин .
- [ ] цвет вершин
- [ ] цвет ребер
- [ ] Программа должна позволять выбирать цвет фона.
- [ ] Настройки должны сохраняться между перезапусками программы.

- [ ] Программа должна позволять сохранять полученные («отрендеренные») изображения в файл в форматах bmp и jpeg.
- [ ] Программа должна позволять по специальной кнопке записывать небольшие «скринкасты» — текущие пользовательские аффинные преобразования загруженного объекта в gif-анимацию (640x480, 10fps, 5s).


### Сущности
model, view, controller, utils tests
